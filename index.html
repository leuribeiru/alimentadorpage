Agendador de Alimentações


Ver código no gitHub

Introdução

O alimentador foi desenvolvido pelos alunos Leonardo, Maiki, Tarlles e Vinicius como trabalho da disciplina de Sistemas Distribuidos do curso Tecnologia em Sistemas para Internet do Instituto Federal de Educação, Ciência e Tecnologia do Sudeste de Minas-Câmpus Barbacena. O alimentador tem a função de gerenciar a alimentação em dois tanques simultaneamente, gerenciando o horário, a quantidade e qual tanque deve ser alimentado.


Cliente

O cliente foi desenvolvido em python com a interface gráfica TKinter. No início da aplicação é necessário informar o endereço do servidor e a chave do servidor de criptografia assimetrica utilizando o método de criptografia AES. O cliente possui as opções de cadastrar uma alimentação informando o horário, a quantidade e qual tanque deve ser alimentado, listar as alimentações que estão agendadas e apagar uma alimentação que está agendada.


Servidor

O servidor possui uma agenda com os horários, quantidade e qual tanque deve ser alimentado, consultas a uma api para fornecimento da hora certa são realizadas frequentemente para garantir que a alimentação seja realizada no horário correto. Quando o servidor determina que a hora certa é igual uma hora da agenda de alimentações uma conexão via bluetooh é estabelecida com o arduino para que a alimentação seja realizada. O servidor também funções de criptografia e arquivos de log informando os horários que as alimentaçoes ocorreram.


Arduíno

O Hardware do projeto é composto por uma placa de prototipagem arduino, um motor servo, um motor corrente contínua, um relé de acionamento de energia, um dispositivo bluetooth. O arduino recebendo os dados corretamente do servidor, efetua operações de controle destes dispositivos. Servo - dispositivo usado para o controle de qual saída de alimentação será utilizada, controlada pelo arduino como direita ou esquerda. Motor Corrente contínua - Controlado pelo acionamento do relé, é utilizado para efetuar o giro da rosca infinita dentro do tubo alimentador empurrando a ração até o orifício de saída. Relé - Efetua controle do motor de corrente contínua, como o cálculo da quantidade de ração e feito em conformidade ao tempo de giro do motor o relé é utilizado para informar quando será ligado e quando será desligado o motor, alimentando ou parando assim uma alimentação. Bluetooth - utilizado para comunicação do arduino com o servidor.
